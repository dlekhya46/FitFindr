# FitFindr ‚Äî Intelligent Candidate Ranker
### Step-by-Step Local Setup & Run Guide

---

## What this system does

Ranks 100,000 candidate profiles against a job description using a three-stage hybrid pipeline:

- **Stage 1** ‚Äî Rule-based hard filter (100K ‚Üí ~34K candidates, ~21s)
- **Stage 2** ‚Äî Multi-signal composite scoring: role fit ¬∑ technical depth ¬∑ behavioral availability ¬∑ context fit (~34K ‚Üí top 500, <1s)
- **Stage 3** ‚Äî Semantic re-ranking via `all-MiniLM-L6-v2` embeddings (top 500 ‚Üí top 100, ~7s)

**Total runtime: ~28 seconds on CPU. No GPU. No network during ranking.**

Output: `submission.csv` ‚Äî 100 ranked candidates with scores and reasoning.

---

## Prerequisites

| Requirement | Minimum | Notes |
|---|---|---|
| Python | 3.10+ | Check: `python3 --version` |
| RAM | 4 GB free | Pipeline peak usage ~1.2 GB |
| Disk | 500 MB free | Dataset + model cache |
| Network | Only for setup | Ranking itself is fully offline |
| GPU | Not required | CPU-only by design |

> **No API keys required.** This system runs entirely locally.
> The only network call is a one-time model download (~23 MB) during setup.

## Step 3 ‚Äî Create a virtual environment

A virtual environment keeps all project dependencies isolated from your system Python.

```bash
python3 -m venv .venv
```

Activate it:

```bash
# macOS / Linux
source .venv/bin/activate

# Windows (Command Prompt)
.venv\Scripts\activate.bat

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

Your terminal prompt will change to show `(.venv)` when activated.

> **Every terminal session:** you must re-run the `source .venv/bin/activate` command each time you open a new terminal window before running any project commands.

## Optional flags for rank.py

```bash
python3 rank.py --help
```

| Flag | Default | Description |
|---|---|---|
| `--candidates` | required | Path to `candidates.jsonl` or `candidates.jsonl.gz` |
| `--out` | required | Output CSV path |
| `--config` | `artifacts/jd_config.json` | JD config file path |
| `--artifacts-dir` | `artifacts` | Directory with `jd_embedding.npy` and `model_cache/` |
| `--top-k` | `500` | Candidates to carry from Stage 2 into Stage 3 |

Example with all flags:

```bash
python3 rank.py \
  --candidates candidates.jsonl \
  --out my_submission.csv \
  --config artifacts/jd_config.json \
  --artifacts-dir artifacts \
  --top-k 500
```

## Troubleshooting

### `ModuleNotFoundError: No module named 'numpy'` (or any other module)

The virtual environment is not activated. Run:
```bash
source .venv/bin/activate
```

### `ERROR: JD config not found: artifacts/jd_config.json`

You are running `rank.py` from a different directory. Always run from the project root:
```bash
cd ~/Desktop/FitFindr
python3 rank.py ...
```

### `[Stage 3] WARNING: artifacts/jd_embedding.npy not found`

Step 5 was not completed. Run:
```bash
python3 generate_jd_artifacts.py
```

### `[Stage 3] Loading all-MiniLM-L6-v2 (may require network...)`

The `artifacts/model_cache/` directory is missing or empty. This means `generate_jd_artifacts.py` was not run, or was run without network access. Re-run with an active internet connection:
```bash
python3 generate_jd_artifacts.py
```

After this, all future runs are fully offline.

### `Submission is valid.` not shown after validator

The validator found format errors and listed them. Common causes:
- Fewer or more than 100 data rows
- A `candidate_id` that does not exist in `candidates.jsonl`
- Scores not monotonically non-increasing by rank

These should not occur with the default pipeline. If they do, file an issue.

### Pipeline takes longer than expected

Ensure PyTorch is the CPU-only build:
```bash
python3 -c "import torch; print(torch.__version__)"
```

If the version ends with `+cu` (e.g., `2.12.1+cu121`), you have the CUDA build installed. Reinstall:
```bash
pip uninstall torch -y
pip install torch --index-url https://download.pytorch.org/whl/cpu
```
